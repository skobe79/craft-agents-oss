# v0.8.15 — Lark / Feishu messaging adapter

## Features

**Lark / Feishu messaging integration (Phases 1 + 2)**

A third messaging platform alongside Telegram and WhatsApp. Connect a Lark
Custom App via the Open Platform (`open.larksuite.com` for international,
`open.feishu.cn` for China) — same `/pair` flow as Telegram, no public
webhook URL required.

Phase 1 (foundation):
- App ID + App Secret + region selector in **Settings → Messaging → Lark / Feishu**
- Long-connection (WebSocket) transport via the official `@larksuiteoapi/node-sdk`
- Text in/out for DMs, plus @mentions in groups (Lark's default scope)
- Session pairing via `/pair <code>`

Phase 2 (parity with Telegram):
- **Live message editing** — assistant responses stream into a single message instead of one bubble per chunk; default `progress` response mode like Telegram
- **Interactive cards** — inline buttons (e.g. plan accept/reject) render as native Lark cards with the first action emphasised; max 10 buttons, 30-char labels
- **Image + file attachments** in both directions; agent files are uploaded via `im.file.create` and sent as `image`/`file` messages (caption follows in a separate text reply)
- **Rich-text formatting** — bold, italic, strikethrough, links, and fenced code blocks render natively via Lark's `post` message type. Plain output keeps the lighter `text` message type so unformatted replies stay one round-trip.

## Setup notes

When configuring the Custom App in Lark Open Platform / Feishu Open Platform:
- Enable scopes: `im:message`, `im:message.group_at_msg`, `im:message:send_as_bot`
- Subscribe to `im.message.receive_v1` event in **Long connection mode** (NOT webhooks)
- Leave the **Encrypt Key** field blank — long connection mode doesn't use webhook payload encryption; filling it in silently breaks inbound events

Full walkthrough: <https://docs.craft.do/messaging/lark>

## Limitations (Phase 1+2)

- Lark and Feishu are separate domains — a bot belongs to one ecosystem, not both
- Bot only receives @mentions in groups (Lark's default `im:message.group_at_msg` scope)
- Edits to messages older than 24 hours are silently dropped (Lark API limit)
- Markdown headers, lists, and tables are sent as plain text within `post`
- Audio, video, and stickers are dropped with a polite log line

## Bug Fixes

- Lark interactive cards now actually render. The card builder was emitting
  the legacy schema-1.0 shape (top-level `elements`) under a `schema: '2.0'`
  envelope, which Lark rejected with code 200621 ("unknown property,
  property: elements"). Wrapped `elements` under `body` to match schema 2.0;
  permission Allow/Deny buttons and Accept Plan buttons now appear in Lark.
- Lark `sendButtons` now logs `lark_send_card_ok` / `lark_send_card_failed`
  with the chat id, button count, and Lark-side error code. On send failure
  it posts a plain-text fallback so the user always sees *something* in
  the chat instead of silent dropped notifications.
- Plan submission flow extended to Lark — was previously hard-gated to
  Telegram only. Lark bindings now get the same Accept plan / Accept &
  compact buttons (with the long-plan `plan.md` attachment) Telegram does.

## Breaking Changes

None. The `lark` platform is fully optional and additive. Existing Telegram
and WhatsApp bindings are untouched.
